  <RadioGroupAdvanced />
</Paper>

```tsx

export default function App() {
```

### AdvancedRadio Props

Here is the API documentation of the AdvancedRadio component. And the rest of the props of AdvancedRadio are the same as the original html input field. You can use props like **value**, **name**, **disabled** etc.

<div className="overflow-x-auto">

|     Props      |          Type           |              Description              | Default |
| :------------: | :---------------------: | :-----------------------------------: | :-----: |
|   className    |        `string`         | Add className to style the container  |  \_ \_  |
| inputClassName |        `string`         | Add inputClassName to style the input |  \_ \_  |
|    children    |       `ReactNode`       |       Pass content as children        |  \_ \_  |
|      ref       | `Ref<HTMLInputElement>` |                                       |  \_ \_  |
|      key       |      `key` `null`       |                                       |  \_ \_  |

</div>

<div className="mb-10" />

### RadioGroup Props

Here is the API documentation of the RadioGroup component.

<div className="overflow-x-auto">

|  Props   |                Type                |          Description          | Default |
| :------: | :--------------------------------: | :---------------------------: | :-----: |
|  value   |              `string`              |        Selected value         |  \_ \_  |
| setValue | `Dispatch<SetStateAction<string>>` | Pass function to select value |  \_ \_  |
| children |            `ReactNode`             |   Radio buttons as children   |  \_ \_  |

</div>
